# Obsidian Vault Structure

Use this structure for a clean Markdown-first vault:

- `00_SYSTEM`
- `01_INBOX`
- `02_IDEAS`
- `03_MEMOS`
- `04_LOGS`
- `05_DECISIONS`
- `06_TASKS`
- `07_SKILLS_PROTOCOLS`
- `99_ARCHIVE`

## Folder Intent

- `00_SYSTEM` holds workflow rules, protocols, and index files.
- `01_INBOX` holds unprocessed notes and captures.
- `02_IDEAS` holds rough concepts and possibilities.
- `03_MEMOS` holds reference notes and working summaries.
- `04_LOGS` holds chronological operational logs.
- `05_DECISIONS` holds durable decision records.
- `06_TASKS` holds task lists and execution tracking.
- `07_SKILLS_PROTOCOLS` holds stable reusable instructions.
- `99_ARCHIVE` holds inactive material.

## Rules

- Keep the vault readable without requiring proprietary plugins.
- Prefer one note per purpose.
- Move unstable draft material out of `01_INBOX` once processed.

## Optional Starter Notes

- `Home` or `Project Map`
- `Instructions`
- `Task Queue`
- `Review Log`

Use plain Markdown links and avoid navigation that depends on local-only workspace state.
