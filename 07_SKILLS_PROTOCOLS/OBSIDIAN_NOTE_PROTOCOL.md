# Obsidian Note Protocol

## Purpose

Keep project notes clean, searchable, and compatible with the repository workflow.

## Note Principles

- Markdown-first
- one clear purpose per note
- stable headings
- easy linking
- reusable structure

## Recommended Note Types

- system notes
- inbox notes
- ideas
- memos
- logs
- decisions
- tasks
- skills and protocols

## Writing Rules

- Use descriptive titles.
- Keep notes concise and structured.
- Link related notes when continuity matters.
- Separate operational logs from durable protocols.
- Keep sensitive data out of the master template vault.

## Navigation Rules

- Prefer one root home or project map note when a derivative repository benefits from a start page.
- Use short folder-level indexes only where they add clarity.
- Prefer plain Markdown links over local workspace UI assumptions.
- Keep note names predictable and stable.

## Compatibility Rules

- Avoid relying on local-only plugin behavior.
- Keep notes readable as plain Markdown.
- Use folders from `TEMPLATE/OBSIDIAN_VAULT_STRUCTURE.md`.

## GitHub Sync Rules

- Track Markdown content and approved support files only.
- Do not track `.obsidian/workspace.json`, `.obsidian/plugins/`, or `.obsidian/themes/`.
- Keep commit scope narrow and descriptive.
- Use pull requests for review-sensitive template changes.
