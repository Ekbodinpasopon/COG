# Obsidian Lite Navigation Protocol

## Purpose

Define a lightweight Obsidian navigation model that does not depend on custom plugins or local UI state.

## Navigation Pattern

- one root home note
- one project map note
- short folder-level indexes where needed
- plain Markdown links
- stable file names

## Rules

- Prefer explicit links over hidden workspace state.
- Keep navigation readable in both Obsidian and plain GitHub rendering.
- Avoid plugin-dependent navigation assumptions.
- Keep folder names and note names predictable.
