# AGENTS.md

## Repository Identity

COG is a master workflow template for `ChatGPT + Codex + Obsidian + GitHub`.

It exists to help start new AI-assisted projects from zero without introducing private project data.

## Scope Rules

- Work only inside the requested template and documentation scope.
- Do not add application code, packages, dependencies, or automation scripts unless explicitly requested.
- Do not turn this repository into a live project repository.
- Keep all outputs Markdown-first and Obsidian-compatible.

## Public Safety Rules

- Use placeholders instead of real project, client, contact, or environment data.
- Do not include real names, emails, phone numbers, URLs, credentials, incidents, prices, or machine-specific paths.
- Do not copy content from private repositories or live operational systems.
- Keep examples sanitized and non-identifying.

## Role Boundaries

### ChatGPT

ChatGPT is the:

- advisor
- planner
- reviewer
- clarifier
- instruction drafter

Use ChatGPT for thinking, planning, reviewing, instruction drafting, and clarification.

### Codex

Codex is the:

- repository executor
- file creator
- file editor
- repository reviewer
- handoff implementer

Use Codex when repository files need to be created, edited, reviewed, organized, committed, or pushed.

### Obsidian

Obsidian is the Markdown knowledge and vault layer.

### GitHub

GitHub is the issue, branch, pull request, and review layer.

## Repository Rules

- Preserve the requested structure.
- Update existing files in place when appropriate.
- Do not delete files unless explicitly requested.
- Do not expand scope beyond the requested workflow template.
- Make assumptions explicit when the repository does not already define them.

## Review Output

After completing work, report only:

1. Files created
2. Files modified
3. Assumptions used
4. Risks / rollback notes
5. Recommended next step
