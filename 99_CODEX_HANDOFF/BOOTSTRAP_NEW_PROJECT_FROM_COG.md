# Bootstrap New Project From COG

Use this prompt when a new repository should be created from the COG template.

## Repository

`[owner/repository]`

## Objective

Set up this repository from the COG template so it can start a new AI-assisted project from zero.

## Scope

- create or update only the Markdown workflow files needed for initial project startup
- preserve the COG operating model
- keep placeholders where project-specific values are not approved

## Files to inspect

- `README.md`
- `AGENTS.md`
- `BRAIN_INDEX.md`
- `00_SYSTEM/`
- `07_SKILLS_PROTOCOLS/`
- `TEMPLATE/`

## Files to create/update

- `[List the exact files to create or update for the new project]`

## Bootstrap checklist

- confirm project identity and scope
- confirm the privacy boundary
- create the root workflow files
- create the initial vault structure
- prepare the first executable issue or handoff

## Constraints

- Markdown-first only
- Obsidian-compatible only
- no application code
- no packages or automation unless explicitly requested
- no redesign beyond the requested structure

## Safety rules

- do not add private names or operational data
- do not copy content from private repositories
- use placeholders until values are approved

## Acceptance criteria

- project root files are present
- vault structure is defined
- issue and handoff workflow is clear
- repository is ready for first executable issue

## Report back format

1. Files created
2. Files modified
3. Assumptions used
4. Risks / rollback notes
5. Recommended next step
