# Master Reference Usage Guide

## Purpose

This repository is a reference layer, not a live operational database.

## How To Use It

- Use `00_SYSTEM/` to define the operating model for a project.
- Use `07_SKILLS_PROTOCOLS/` to standardize recurring AI and repository workflows.
- Use `TEMPLATE/` files as starting points for new project artifacts.
- Use `99_CODEX_HANDOFF/` prompts to hand clear work to Codex.
- Use `EXAMPLES/` only as sanitized formatting references.

## Control Rules

- Keep template content generic.
- Move real project details into the derived project repository only after approval.
- Replace placeholders intentionally and track the decision in project instructions.
- Re-run publication safety review before moving any derivative repo to public visibility.
